404-Error-Handling with TYPO3

The variables for the 404 Error-Handling are saved into the file typo3conf/urltoolconf_404.php.
include(PATH_typo3conf.'urltoolconf_404.php'); is added to typo3conf/AdditionalConfiguration.php

    [FE][pageNotFound_handling]
    Strings: redirect URL, eg. 'notfound.html' or 'http://www.domain.org/errors/notfound.html'.

    If prefixed with "READFILE:" then it will expect the remaining string to be a HTML file which will be read and outputted directly after having the marker "###CURRENT_URL###" substituted with REQUEST_URI and ###REASON### with reason text,
    for example: "READFILE:fileadmin/notfound.html".

    [FE][pageNotFound_handling_statheader]
    If 'pageNotFound_handling' is enabled, this string will always be sent as header before the actual handling.

    More information ...
    - Install-Tool (All configuration)
    - http://www.typo3.net/backendkonfiguration/frontend_konfiguration/#pageNotFound_handling

    Attention: This extension updates the file AdditionalConfiguration.php of your TYPO3-Installation!

RealUrl-Configuration

Settings for the Realurl-Configuration are saved into the file typo3conf/urltoolconf_realurl.php.
include(PATH_typo3conf.'urltoolconf_realurl.php'); is added to typo3conf/AdditionalConfiguration.php

more information:

    - realurl documentation
    - realurl documentation (legacy)

    Attention: This extension updates the file AdditionalConfiguration.php of your TYPO3-Installation!


Miscellaneous

	- Flushing the system cache also flushes the realurl cache (maybe you have to enable system cache flushing in the install tool). This affects urldata, not pathdata.
	- 404 error handling works now with multiple languages (when in page mode)
	- since realurl 2.0 there is no error log anymore
	- you can use umlauts and alike in URLs, but be aware that this is experimental. Enable this feature in the extension manager configuration. Please contact me, when you run into any issues.
